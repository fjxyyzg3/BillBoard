# 新增记录成功提示轻量化设计

## 背景

新增记录页当前在保存成功后，在表单内部显示绿色成功区域，并提供“再记一笔”和“返回首页”两个链接。由于用户保存后仍停留在新增记录页，可以继续录入；底部 tab 也已经提供返回首页能力，因此这两个按钮是冗余交互。

## 假设

- 保存成功后继续停留在新增记录页。
- 成功提示只需要确认保存结果，并展示记录类型与金额。
- 提示不应改变表单布局，避免打断连续录入。
- 自动消失时间为 2.5 秒。

## 方案

采用顶部弹出式轻量 toast。保存成功后，复用现有 URL 参数中的 `created`、`type` 和 `amountFen` 生成提示内容。提示显示“记录已保存”和“支出：78.00”或“收入：4,321.09”等详情，短暂停留后自动隐藏。

不引入全局 toast 系统，不新增导航按钮，不修改保存流程和数据写入逻辑。

## UI 行为

- toast 固定显示在视口顶部，覆盖在当前页面之上，不占据表单空间。
- toast 保持轻量视觉层级，避免原绿色大面板的重量感。
- toast 出现后 2.5 秒自动隐藏。
- 如果保存后出现新的 action 错误状态，则不展示成功 toast。

## 代码影响

- `src/components/transaction-form.tsx`：将成功面板替换为顶部 toast，移除成功态里的 `Link` 按钮渲染，并增加自动隐藏状态。
- `src/app/(app)/add/page.tsx`：继续保留现有成功参数读取逻辑，无需改动或只做必要的 prop 清理。
- `tests/e2e/create-expense.spec.ts`、`tests/e2e/create-income.spec.ts`：更新成功反馈断言，确认两个冗余链接不再出现，并继续验证记录可在记录页看到。
- `tests/integration/create-transaction.test.ts`：更新源码级 UI 断言，确认成功反馈使用轻量 toast，不再要求后续操作链接。

## 验证目标

1. 保存支出后，页面显示轻量成功提示，包含保存成功、类型和金额。
2. 保存收入后，页面显示轻量成功提示，包含保存成功、类型和金额。
3. 成功反馈不再提供“再记一笔”和“返回首页”按钮。
4. 用户仍可通过底部 tab 进入记录页，并看到刚保存的记录。
5. 最小验证先运行相关 E2E；若后续需要提交，再按仓库规则运行完整验证。
